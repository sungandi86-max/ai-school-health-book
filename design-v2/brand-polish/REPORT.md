# Book v2 Brand Polish — Sample Review

`book-v2-brand-samples.pdf` (18페이지, 12개 대표 위치 샘플)를 기준으로 합니다. 전체 250페이지는 재조판하지 않았습니다. 원고 문장·Chapter/PART 순서·QR·이미지·기본 색상 체계(딥네이비+연블루)·레이아웃 구조는 변경하지 않았습니다.

## 수정한 대표 페이지

**차례 (구 2페이지)** — 기존 문제: 없음(구조 유지). 수정 원칙: 손대지 않음. 수정 내용: 없음. 유지한 요소: 전체. 시각적 차이: 없음.

**전체 Workflow (구 4페이지)** — 기존 문제: 없음(지시대로 구조 유지). 수정 원칙: "4페이지의 전체 Workflow는 현재 구조 유지". 수정 내용: 없음(단, 러닝 헤더 버그 수정으로 이 페이지에 더 이상 "에필로그"가 잘못 표시되지 않음 — 아래 별도 항목 참고). 유지한 요소: 포스터 이미지, 캡션, 클로징 문장. 시각적 차이: 러닝 헤더 오표시 제거.

**PART 1 표지 (구 5페이지)** — 기존 문제: 흰 배경과 텍스트만 있어 PART 전환 감각이 약함. 수정 원칙: 대형 타이포 + 핵심 동사 키워드 + 수달 시그니처. 수정 내용: "PART 01"을 52pt 연블루 굵은 활자로 배치, 그 아래 PART별 핵심 동사("발견하다") 병기, 하단에 수달 시그니처 1회 배치. 유지한 요소: 원고 제목("왜 보건실은 늘 바쁠까"), Chapter 범위 표기, 태그라인, 여백 중심 구조. 시각적 차이: 텍스트 전용 → 큰 타이포+키워드+시그니처가 있는 표지로 전환, PART가 바뀌는 순간이 분명해짐.

**Chapter 01 오프닝 (구 6페이지, A형·이야기형)** — 기존 문제: 제목 아래 넓은 빈 공간(220pt 스페이서)만 있고 하단 진행 카드까지 의미 없는 여백이 반복됨. 수정 원칙: "하단 카드 위에 의미 없는 빈 공간만 크게 남기지 않는다". 수정 내용: 제목 위에 짧은 장식선을 추가하고 하단 여백을 220→90pt로 축소, 진행 카드가 제목과 더 가깝게 붙도록 조정. 유지한 요소: 큰 제목, 진행 카드(오늘 만드는 것/시간/난이도), 진행 상태 표기. 시각적 차이: "제목 - 큰 여백 - 카드"였던 구조가 "제목 - 짧은 여백 - 카드"로 압축되어 한 페이지 안에서 시선이 자연스럽게 이어짐.

**핵심 정리 (구 10페이지)** — 기존 문제: 카드 안 모든 문장이 비슷한 굵기·크기라 어떤 문장이 핵심인지 한눈에 구분되지 않음. 수정 원칙: "문장 하나가 중심이 되도록 카드 높이를 줄이고, 굵은 문장은 더 크게". 수정 내용: 라벨을 작은 eyebrow(8pt)로 축소하고, 마지막 굵은 문장만 11.2pt 볼드 네이비로 확대, 카드 패딩을 13→11로 축소. 유지한 요소: 카드 배경색(연블루), 원고 문장. 시각적 차이: 카드 안에서 시선이 마지막 문장으로 바로 꽂힘.

**실습 (구 12페이지)** — 기존 문제: 없음(사용자 지적 대상 아님). 수정 내용: 없음(표·체크리스트·진행 카드 구조 그대로).

**Chapter 마무리 (구 13페이지)** — 기존 문제: 쑤캥의 한마디·오늘 바뀐 것·다음 Chapter Preview가 모두 같은 연블루 카드라 성격이 구분되지 않음. 수정 원칙: 저자 노트/대비 강조/가벼운 마무리 장치로 각각 분리. 수정 내용: (1) 쑤캥의 한마디에 수달 시그니처 아이콘 + 왼쪽 골드톤 세로선 추가, (2) 오늘 바뀐 것의 After 칸에만 흰 배경 강조 박스를 얹어 Before(회색)-After(굵은 네이비) 대비 강화, (3) 다음 Chapter Preview를 채워진 카드 대신 얇은 상단 선 + 회색 라벨 형태로 전환. 유지한 요소: 카드 기본 색상 체계, 문장 내용. 시각적 차이: 세 컴포넌트가 한눈에 서로 다른 성격으로 읽힘.

**QR·프로젝트 완료 (구 21페이지)** — 기존 문제: PROMPT/완료/오늘 만든 프로젝트/QR 네 블록이 연속으로 쌓여 카드 벽처럼 보임. 수정 내용: PROMPT는 기존 흰 배경 테두리 스타일 유지(다른 카드와 이미 구분됨). 나머지 완료 표시·진행 카드·QR 카드는 이번 12페이지 범위에서는 카드 자체 구조를 바꾸지 않음. 시각적 차이: 크지 않음 — **아직 결정이 필요한 항목에 별도 기재**(아래 참고).

**CASE (구 28페이지)** — 기존 문제: Workflow·핵심 메시지와 같은 연블루 카드라 "기록 카드"라는 성격이 드러나지 않음. 수정 원칙: 왼쪽 세로선 + 번호형 구조 + 약간 더 또렷한 배경. 수정 내용: 카드 배경을 연블루보다 살짝 진한 톤(CASE_BG)으로 바꾸고 왼쪽에 3pt 파란 세로선을 추가, "사람이 한 일/도움을 받은 일/다시 사람이 한 일" 세 항목에 01/02/03 번호를 매김(카드 자체의 CASE 제목 줄은 번호에서 제외). 유지한 요소: 원고 문장, 카드 형태(둥근 모서리). 시각적 차이: "사례 기록표"처럼 구조화되어 Workflow·핵심 메시지와 뚜렷이 구분됨.

**Workflow 본문 카드 (구 37페이지)** — 기존 문제: 단순 "↓" 문자만 나열되어 화살표 구분이 약하고 CASE와 시각적으로 거의 동일함. 수정 원칙: 흐름이 먼저 보이도록, 일반 박스보다 넓은 내부 여백, 단계 번호. 수정 내용: 각 단계에 01~04 번호를 붙이고 "↓"를 24pt 남색 원형 칩으로 확대, 카드 왼쪽에 3pt 네이비 세로선(CASE의 파란 세로선과 색을 달리해 구분) 추가, 카드 내부 패딩 12→18. 유지한 요소: 원고 문장, Workflow 카드가 다루는 실제 단계 내용. 시각적 차이: 화살표가 압도적으로 커지고 번호가 붙어 "다이어그램/지도"처럼 보임.

**실제 프로젝트 화면 (구 47페이지)** — 기존 문제: 캡션과 이미지 사이 위계가 불명확하고 "실제 화면"이라는 성격이 라벨 없이 제시됨. 수정 원칙: 이미지가 주인공, 캡션-이미지 위계 명확화, 테두리/그림자 최소화. 수정 내용: 이미지 위에 "실제 화면" eyebrow 라벨 추가, 이미지 폭 94%→96%로 확대, 캡션 spaceBefore 축소로 이미지와 더 밀착. 유지한 요소: 이미지 원본, IMG 내부 표기 비노출 원칙, 캡션 문장. 시각적 차이: 이미지가 더 크고 명확한 하나의 장면으로 인식됨.

**PART 마무리 (구 72페이지)** — Chapter 마무리와 동일한 세 가지 수정(쑤캥의 한마디/오늘 바뀐 것/다음 PART Preview)이 그대로 적용됨. 시각적 차이: Chapter 마무리와 마찬가지로 세 컴포넌트가 명확히 구분됨.

## Chapter 오프닝 3종

같은 디자인 시스템(화이트 배경, 네이비 제목, 진행 카드 구조) 안에서 다음 3가지로만 변형했습니다.

- **A형(이야기형)** — Chapter 1·2·5·6·7. 제목 위 짧은 장식선, 여백을 충분히 주되 하단은 압축. 감성 에세이 오프닝에 가까움.
- **B형(구조·판단형)** — Chapter 3·4·8~16. 제목 왼쪽에 작은 정사각 포인트(담백하고 구조적인 느낌), 여백을 A형보다 더 좁게.
- **C형(AI·운영형)** — Chapter 17~22. 제목 아래 "사람 → AI → 데이터 → 시스템" 구조 스트립(연블루 칩)을 배치해 운영/자동화 성격을 시각적으로 강조. 이 네 단어는 원고 전체에서 이미 반복적으로 쓰이는 개념어이며 새로 지어낸 문장이 아닙니다.

12페이지 샘플에는 A형(Chapter 01)만 실제로 포함되어 있습니다. B형·C형은 코드로 구현했지만 이번 샘플 세트에는 대표 예시가 없어 별도 확인이 필요합니다(아래 "아직 결정이 필요한 항목" 참고).

## PART 표지 체계

8개 PART 전부에 다음 키워드를 부여했습니다(원고 제목은 그대로 두고, 표지에서만 보조적으로 병기).

PART 1 발견하다 · PART 2 경계를 그리다 · PART 3 실패를 기록하다 · PART 4 구조를 만들다 · PART 5 흐름을 연결하다 · PART 6 운영으로 확장하다 · PART 7 반복을 맡기다 · PART 8 시스템으로 남기다

## CASE / Workflow / 핵심 메시지 차이

| 구분 | 배경 | 구조 장치 | 성격 |
|---|---|---|---|
| CASE | 연블루보다 살짝 진한 톤 | 왼쪽 파란 세로선 + 01/02/03 번호 | 기록 카드 |
| Workflow | 기존 연블루 | 왼쪽 네이비 세로선 + 원형 화살표 칩 + 넓은 여백 | 흐름/다이어그램 |
| 핵심 메시지 | 기존 연블루 | eyebrow 라벨 + 확대된 볼드 문장 | 한 문장 강조 |

## 쑤캥 브랜드 요소 사용 위치

`assets/07_ai_workflow.png`에 이미 그려져 있던 수달 그림을 그대로 잘라 쓴 것으로(`assets_derived/otter_signature.png`), 새 캐릭터를 생성하지 않았습니다. 12페이지 샘플에서는 PART 표지, 쑤캥의 한마디(2회), 작은 축하(코드 구현, 샘플에는 미포함) 위치에 사용했습니다. **250페이지 전체로 확장할 경우 매 쑤캥의 한마디마다 노출하면 안 되므로(전체 8~15회 제한), 노출 위치를 좁히는 규칙이 추가로 필요합니다** — 아래 "아직 결정이 필요한 항목" 참고.

## 가독성 유지 여부

본문 9.6pt, 이야기 본문 10pt, 행간·캡션·Prompt·체크리스트 크기는 전혀 건드리지 않았습니다. 글자를 키워서 문제를 해결한 곳은 없으며, 카드가 답답해 보일 수 있는 CASE·Workflow는 패딩을 넓히는 방식으로만 대응했습니다.

## 전체 책 확장 시 예상 페이지 수 변화

- PART 표지: 키워드 한 줄과 수달 이미지 한 장이 추가되지만 표지는 원래도 한 페이지 전용이라 페이지 수 변화 없음(8곳).
- Chapter 오프닝: 하단 여백을 줄였으므로 오히려 진행 카드가 같은 페이지에 더 안정적으로 들어가 페이지 수가 늘어날 요인은 없음(22곳).
- CASE/Workflow 카드: 내부 패딩이 늘어난 만큼(Workflow 카드 6pt 증가) 카드 자체 높이가 소폭 커짐 — 카드가 페이지 하단에 걸쳐 있던 극소수 지점에서 다음 페이지로 밀릴 가능성이 있음(전체 250페이지 기준 수 페이지 이내로 추정, 실제 재조판 전에는 정확히 알 수 없음).
- 종합 예상: ±5페이지 이내(250페이지 기준 245~255페이지 범위로 추정). 실제 수치는 전체 재조판 전에는 추정치입니다.

## 부가 발견 및 수정 — 러닝 헤더 버그

12페이지 샘플 작업 전, 최근 커밋된 `design-v2/fullbook/book-v2-final.pdf`를 재확인하는 과정에서 사용자가 지적한 문제를 실제로 확인했습니다: 2~4페이지(차례, 전체 Workflow) 상단에 "에필로그"라는 잘못된 러닝 헤더가 표시되고 있었습니다. 원인은 러닝 헤더용 상태값(`CURRENT`)이 문서 조립 단계에서 동기적으로도 갱신되고 있어, 실제 페이지가 그려지는 시점에는 이미 "마지막으로 처리된 챕터"(에필로그)의 값으로 고정되어 있었기 때문입니다. 조립 단계 전용 상태(`BUILD_STATE`)를 분리해 `CURRENT`는 오직 실제 렌더링 시점에만 갱신되도록 고쳤습니다. 이번 `brand_build.py`에는 이 수정이 반영되어 있으며, 차례·Workflow 페이지는 이제 러닝 헤더가 아예 표시되지 않습니다(사용자가 제시한 두 옵션 중 "헤더 없음"에 해당).

## 아직 결정이 필요한 항목

1. **QR·프로젝트 완료 페이지(구 21페이지)의 카드 벽 현상** — PROMPT/완료/오늘 만든 프로젝트/QR 네 블록이 여전히 연속으로 쌓입니다. 이번 12페이지에서는 손대지 않았으나, 카드 사이 간격을 넓히거나 일부를 얇은 선 스타일로 전환하는 추가 작업이 필요한지 확인이 필요합니다.
2. **수달 시그니처 노출 빈도 규칙** — 전체 250페이지로 확장할 때 "쑤캥의 한마디"마다 노출하면 8~15회 제한을 크게 초과합니다(원고에 쑤캥의 한마디가 약 20회 이상 존재). PART별 1회, 또는 특정 Chapter 번호만 선별하는 화이트리스트 규칙이 필요합니다.
3. **Chapter 오프닝 B형·C형 실제 샘플 확인** — 이번 12페이지에는 A형(Chapter 01)만 포함되어 있어, B형(구조·판단형)·C형(AI·운영형)이 실제 원고 제목과 함께 어떻게 보이는지 별도로 확인이 필요합니다.
4. **주의(Warning) 카드** — 코드는 옅은 웜/코랄 톤으로 수정했지만 12페이지 샘플에는 해당 컴포넌트가 없어 포함되지 않았습니다(별도 테스트 렌더링으로 확인함, 보고서에는 캡처 없음).

전체 250페이지 재조판은 위 항목에 대한 결정과 사용자 승인 없이는 진행하지 않습니다.

# Round 2 — 출판사 최종 편집디자이너 수준 폴리싱

같은 12개 대표 페이지(`book-v2-brand-samples.pdf`, 18페이지)에 대해 "새 디자인"이 아니라 "디자인 폴리싱"만 진행했습니다. 원고 내용·문장·Chapter/PART 제목·Workflow 구조·프로젝트 화면·이미지·QR·색상 체계(딥네이비+연블루)·전체 레이아웃 구조·페이지 순서는 이번에도 변경하지 않았습니다. 작업 중 사용자가 첨부한 『보건실은 운영된다』 참고 PDF를 다시 펼쳐 3·5·7페이지의 여백 감각과 정보 위계를 참고했습니다 — 다만 그 책의 "본문 전체를 하나의 큰 카드에 담는" 구조는 우리 책의 "특수 컴포넌트에만 카드를 쓰는" 전체 레이아웃 원칙과 달라 그대로 가져오지 않았고, PART 표지의 절제된 여백감·타이포 대비만 판단 기준으로 참고했습니다.

## 1. 수정한 페이지

- **PART 1 표지 (구 5페이지)** — PART 번호를 52pt → 132pt로 대폭 확대하고 얇은 3pt 네이비 룰과 자간을 준 "PART 01" 라벨을 추가해 번호 자체가 표지의 중심 그래픽 요소가 되도록 재구성.
- **QR·프로젝트 완료 (구 21페이지)** — "오늘 만든 프로젝트" 블록을 채워진 카드에서 얇은 상단 선 + eyebrow 라벨 형태로 전환해 PROMPT/QR 카드 사이 "카드 벽" 현상을 완화(QR·PROMPT 박스는 사용자 지시대로 유지).
- **실습 (구 12페이지)** — 원고 그대로 비어 있는 "나의 하루 한 장면" 표를 예시 표와 다른 스타일(행 높이 확대, 연블루 톤 배경, "WRITE HERE · 직접 적어보세요" 안내 문구)로 렌더링해 워크북처럼 보이도록 조정.
- **Chapter 마무리 / PART 마무리 (구 13, 72페이지)** — 쑤캥의 한마디를 웜 크림 배경 + 큰 장식 따옴표 + "AUTHOR'S NOTE" eyebrow + 수달 아이콘 재배치로 재구성. 오늘 바뀐 것을 가로 3단 표에서 세로 Before↓After 스택 구조로 전면 재설계(Before는 8.4pt 회색, After는 12.4pt 볼드 네이비 하이라이트 박스)하고, 카드 안에 있던 "오늘 바뀐 것" 라벨이 바로 위 원고 제목과 중복 표시되던 것을 발견해 제거.
- **PROMPT 라벨, EPILOGUE 표지 라벨** — 자간 라벨 체계를 통일하기 위해 추가로 자간 처리.

## 2. 수정 이유

사용자가 지적한 8개 항목(PART 표지 임팩트/카드 벽 제거/실습 워크북화/쑤캥 브랜드성/오늘 바뀐 것 대비/페이지 리듬/타이포 위계/절제 원칙) 중 실제로 손댈 수 있는 구조 변경 없이 적용 가능한 항목을 12페이지 샘플에 반영했습니다. 특히 "오늘 바뀐 것" 대비는 사용자가 제시한 세로 ASCII 목업(Before → 화살표 → After)을 그대로 코드 구조로 옮겼습니다.

## 3. Before → After 비교

| 항목 | Before (Round 1) | After (Round 2) |
|---|---|---|
| PART 표지 | 52pt 번호, 보조 요소 수준 | 132pt 번호, 표지의 주인공 그래픽 |
| 오늘 바뀐 것 | 가로 3단, After만 흰 배경 강조 | 세로 스택, Before 8.4pt 회색 → After 12.4pt 볼드 네이비 박스, 대비 극대화 |
| 오늘 만든 프로젝트 | 연블루 채움 카드 | 얇은 선 + eyebrow 라벨(카드 없음) |
| 쑤캥의 한마디 | 일반 카드 + 작은 아이콘 | 웜 크림 배경 + 큰 따옴표 + 자간 라벨 + 아이콘 |
| 실습 빈 표 | 예시 표와 동일한 스타일 | 행 높이 확대 + 톤 배경 + "WRITE HERE" 안내 |
| 자간(letter-spacing) 라벨 전체 | 코드는 있었지만 폰트에 없는 U+2009(얇은 공백) 글리프를 써서 **실제로는 자간이 전혀 적용되지 않던 버그** | 일반 스페이스를 인라인 폰트 크기로 축소하는 방식으로 교체 — PART 01/AUTHOR'S NOTE/PROMPT/WRITE HERE/EPILOGUE 전부 실제로 자간이 보임 |

## 4. 아직 부족한 점

- Round 1에서 남겨둔 미해결 항목 중 다음은 이번 라운드에서도 다루지 않았습니다: Chapter 오프닝 B형(구조·판단형)·C형(AI·운영형)이 12페이지 샘플 안에 실제 예시로 없다는 점, 수달 시그니처를 250페이지 전체로 확장할 때 필요한 노출 빈도 화이트리스트 규칙, 주의(Warning) 카드가 이번 샘플 범위에 없다는 점.
- "카드 벽" 완화는 QR·프로젝트 완료 페이지의 절반(오늘 만든 프로젝트)에만 적용했고, PROMPT·QR 박스는 사용자 지시대로 의도적으로 유지했습니다 — 이것이 실제로 "카드 벽이 해소되었다"고 보기에 충분한지는 250페이지 전체에서 반복 노출 빈도를 봐야 확정할 수 있습니다.
- 타이포그래피 위계는 이번 라운드에서 자간 버그 수정과 라벨 체계 통일에 집중했고, `S` 딕셔너리 전체의 크기·행간 값을 처음부터 다시 설계하지는 않았습니다 — 기존 값들이 이미 여러 차례 조정을 거쳤다고 판단해 추가 변경의 위험(회귀) 대비 이득이 낮다고 보았습니다.
- 12페이지 샘플 기준의 판단이며, 250페이지 전체로 확장했을 때 카드 높이 변화로 인한 페이지 수 변동(Round 1에서 추정한 ±5페이지)은 이번 라운드 변경분(오늘 바뀐 것 세로 스택화로 카드가 다소 길어짐, 실습 표 행 높이 확대)까지 더하면 재추정이 필요합니다.

## 5. 출판사 최종 QA 기준 평가

12개 대표 페이지 자체는 PART 표지·오늘 바뀐 것·쑤캥의 한마디·실습 페이지에서 뚜렷한 개선이 있었고, 자간 버그처럼 이번에 처음 발견해 고친 실제 결함도 있었습니다. 다만 위 "아직 부족한 점"에 적어둔 대로 Round 1부터 이어진 미해결 항목(B/C형 Chapter 샘플, 수달 노출 규칙, Warning 카드)이 여전히 남아 있고, 이 12페이지만으로는 250페이지 전체에서 페이지 리듬이 실제로 "서점 실용서 수준"으로 유지되는지 확인할 수 없습니다. **전체가 만족스럽지 않으면 절대로 "출간 가능"이라고 말하지 말라는 지시에 따라, 이번 단계에서도 "출간 가능"이라고 말하지 않습니다.** 남은 항목에 대한 결정과 확인 없이는 250페이지 전체 재조판으로 넘어가지 않는 것을 권합니다.

# Round 3 — Final Polish (읽는 경험 완성 단계)

이번 라운드는 디자인을 새로 만들거나 바꾸는 단계가 아니라, 이미 확정된 book-v2-final 디자인 위에서 "읽는 경험"만 다듬는 단계로 진행했습니다. 원고·문장·문체·Chapter/PART·페이지 순서·Workflow 구조·이미지·프로젝트 화면·QR·색상 시스템(딥네이비+연블루)·전체 레이아웃 구조는 이번에도 변경하지 않았습니다. 새 카드·새 색·새 아이콘·새 장식·폰트 교체·애니메이션 느낌은 추가하지 않았고, 오히려 기존 요소의 정렬·간격·위계만 덜어내고 정리하는 방향으로 작업했습니다.

## 1. 수정한 페이지

- **Workflow 본문 카드 (구 37페이지)** — 화살표 칩의 위치, 번호-본문 칸 너비 계산.
- **실제 프로젝트 화면 (구 47페이지)** — 이미지 프레임(매트+테두리).
- **Chapter 01 오프닝 (구 6페이지, A형 — B/C형도 코드상 동일 원리 적용)** — 제목 뒤 여백 리듬.
- **전체 Workflow 페이지 (구 4페이지)** — 핵심 문장("이 책은 이 Workflow를 하나씩 완성하는 과정입니다") 위계.
- **본문 스타일 전역(S["body"])** — 문단 간격(spaceAfter). 12개 샘플 중 본문이 있는 모든 페이지(구 10·13·37·47·72페이지 등)에 적용됨.
- 그 외 10개 샘플 페이지는 이번 라운드에서 직접 수정하지 않았으나, PART 표지·쑤캥의 한마디·오늘 바뀐 것이 Chapter 마무리(구 13)와 PART 마무리(구 72)에서 동일한 규칙으로 나타나는지 재확인했습니다(아래 4번 참고).

## 2. 수정 이유

① Workflow가 "카드"처럼 보인다는 지적에 따라, 화살표 칩을 카드 전체 폭 기준 가운데가 아니라 번호 배지 칸 바로 아래로 옮겨 번호-화살표-번호가 하나의 세로 축으로 이어지도록 정렬만 바꿨습니다. 카드 배경색은 그대로 두었습니다(색 추가 없음).
② 프로젝트 화면이 "스크린샷을 그냥 붙여넣은" 느낌이라는 지적에 따라, 허용된 범위(아주 얇은 그림자, 아주 연한 테두리) 안에서 화면을 중립 회색 매트 위에 얹은 형태로 감쌌습니다. 이미지 파일·캡션 문장은 손대지 않았습니다.
③ Chapter 오프닝이 "심심하다"는 지적에 따라, 제목과 태그라인/진행 카드 사이의 단일하고 거대한 여백(90pt)을 절반 가까이로 줄여 시선이 제목에서 태그라인·진행 카드로 자연스럽게 흐르도록 리듬을 조정했습니다. 새 장식 요소는 추가하지 않았습니다.
④ 전체 Workflow 페이지가 "책의 얼굴"이라는 지적에 따라, 마지막 핵심 문장의 크기(10.6→11.8pt)와 위쪽 여백(spaceBefore 0→10)만 키워 이 문장이 페이지의 결론처럼 읽히게 했습니다. 처음에는 13pt·spaceBefore 20으로 시도했으나 문장이 다음 페이지로 밀려나 거의 빈 페이지가 되는 부작용이 있어(qpdf 페이지 수 18→19 확인 후) 같은 페이지 안에 자연스럽게 남는 값으로 재조정했습니다.
⑤ 30분 이상 읽어도 편안하도록 본문 문단 간격을 10→11.5pt로 아주 소폭 늘렸습니다. 행간(leading)은 이미 9.6pt 기준 18.8pt(약 1.96배)로 상당히 여유로운 상태였다고 판단해 추가로 늘리지 않았고, 폰트 크기도 그대로 두었습니다.

## 3. Before → After 비교

| 항목 | Before | After |
|---|---|---|
| Workflow 화살표 위치 | 카드 전체 폭 기준 가운데 정렬(번호 칼럼과 어긋남) | 번호 배지 칸(26pt) 바로 아래로 정렬 — 번호·화살표가 하나의 세로 축을 이룸 |
| 프로젝트 화면 | 배경 위에 이미지만 놓임(순수 스크린샷 느낌) | 옅은 회색 매트 + 0.75pt 연한 테두리로 감싸 "지면에 배치된 실제 화면"처럼 보임 |
| Chapter 오프닝(A형) | 제목 뒤 90pt 단일 공백 → 태그라인 | 제목 뒤 54pt로 압축, 태그라인·진행 카드로 이어지는 리듬 |
| 전체 Workflow 핵심 문장 | 10.6pt, 캡션과 간격 없이 바로 이어짐 | 11.8pt, 위쪽 여백 10pt로 살짝 분리되어 결론처럼 읽힘 |
| 본문 문단 간격 | spaceAfter 10pt | spaceAfter 11.5pt |

## 4. 독서 경험이 어떻게 좋아졌는지

Workflow 카드는 이제 번호와 화살표가 같은 세로선 위에 있어 눈이 위에서 아래로 미끄러지듯 따라가게 되어, "설명을 읽는다"보다 "순서를 따라간다"는 느낌에 더 가까워졌습니다. 프로젝트 화면 페이지는 매트 프레임 덕분에 화면이 지면 위에 신중하게 놓인 하나의 "장면"처럼 보이고, 본문 텍스트와의 위계가 더 또렷해졌습니다. Chapter 오프닝은 제목을 읽은 뒤 바로 다음 요소(태그라인·오늘 만들 것)로 시선이 이어져 페이지 중간에서 시선이 끊기는 느낌이 줄었습니다. 전체 Workflow 페이지는 마지막 문장이 살짝 더 크고 살짝 더 떨어져 있어 "책을 펼치자마자 이해해야 할 한 문장"이라는 역할이 분명해졌습니다. 본문은 문단 사이 공백이 아주 조금 늘어 문장 덩어리가 조금 더 여유 있게 나뉘어 보입니다.

## 5. 아직 남아 있는 아쉬운 점

- 이번 라운드는 12페이지 샘플(Chapter 01=A형)만 렌더링해 확인했습니다. Chapter 오프닝 B형·C형, 그리고 22개 Chapter 전체에서 여백 리듬 조정이 실제로 어떻게 보이는지는 전체 재조판 전에는 확인할 수 없습니다.
- 본문 문단 간격(spaceAfter) 조정은 이 12페이지에서는 페이지 수 변화가 없었지만, 250페이지 전체에 적용하면 문단이 많은 페이지에서 누적된 간격 증가로 일부 페이지가 다음 장으로 밀릴 가능성이 있습니다(정확한 수치는 전체 재조판 전에는 추정치입니다).
- Round 1·2에서 남겨둔 미해결 항목(Chapter B/C형 실제 샘플, 수달 시그니처 노출 빈도 규칙, 주의(Warning) 카드 미포함)은 이번 라운드에서도 다루지 않았습니다.
- "카드 벽" 완화는 Round 2에서 이미 적용한 정도(오늘 만든 프로젝트만 얇은 선으로 전환) 그대로이며, 이번 라운드에서 추가로 손대지 않았습니다.
- 프로젝트 화면의 매트 프레임과 그림자 효과는 ReportLab이 실제 블러(blur) 그림자를 지원하지 않아 "여백 차이로 그림자를 흉내 낸" 근사치입니다. 인쇄 시 실제 종이 위에서는 화면 대비 미묘하게 다르게 보일 수 있어, 실제 인쇄본 교정지에서 한 번 더 확인하는 것을 권합니다.

## 6. 출판사 최종 QA 기준 평가

이번 라운드에서 다룬 5개 항목(Workflow 다이어그램화, 프로젝트 화면 프레이밍, Chapter 오프닝 리듬, 전체 Workflow 핵심 문장 위계, 본문 문단 간격)은 모두 절제된 범위 안에서 실제로 "읽는 경험"이 개선되었다고 판단합니다. 다만 이 12페이지 샘플만으로는 250페이지 전체, 특히 Chapter B형·C형 22개 전부에서 같은 리듬 조정이 안전하게 적용되는지, 문단 간격 변경이 페이지 수에 미치는 누적 효과가 어느 정도인지 확인할 수 없습니다. 브랜드 4요소(PART/Workflow/쑤캥의 한마디/오늘 바뀐 것)는 각각 단일 함수로 렌더링되는 구조라 코드 구조상 전체 챕터에 동일하게 적용되지만, 실제 화면으로는 Chapter 마무리(구 13)와 PART 마무리(구 72) 두 곳에서만 육안 확인했습니다.

## 7. 출간 가능 여부

**출간 가능이라고 말하지 않습니다.** 이번 라운드의 개별 개선은 방향이 맞다고 판단하지만, 지시하신 대로 "조금이라도 마음에 걸리는 부분이 있으면 출간 가능이라고 하지 말라"는 기준에 따라 — 위 5번 항목의 미확인 영역(전체 250페이지 적용 시 페이지 수 변화, Chapter B/C형 실제 확인, Round 1·2의 미해결 항목)이 남아 있는 한 이 단계에서 "서점에 진열되어도 어색하지 않다"고 단정할 수 없습니다. 전체 250페이지 재조판과 실제 인쇄본 교정 확인 없이는 최종 출간 가능 판단을 내리지 않는 것을 권합니다.
