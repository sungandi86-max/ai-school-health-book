# design-v2/fullbook — 전체 255페이지(상당) 조판본 (교정쇄 단계)

승인된 Book Design v2 대표 페이지 6장의 디자인을 `book-source/book-source-final.md` 전체(PART 1~8,
Chapter 01~22, 에필로그, 실습, Workflow, CASE, 프로젝트 화면, QR)에 그대로 확장 적용한 전체 조판본입니다.
새로운 디자인이 아니라 승인된 디자인 시스템의 일관 적용이며, 기존 `pdf/book-final-pass2.pdf`와
그 조판 코드는 참고하지 않았습니다.

## 파일

- `book-v2-final.pdf` — 전체 조판 PDF (250페이지, A5)
- `full_build.py` — 전체 조판 스크립트 (`design-v2/design_v2.py`의 승인된 스타일 값을 그대로 이식,
  파서는 기존 `parser.py`의 순수 구조 파서를 재사용)
- `fonts/` — 본문에 사용된 서체(Noto Sans CJK KR 서브셋, 전체 원고 + 스크립트 문자 기준 재추출)

## 승인된 디자인과의 관계

- 색상 시스템, 카드 스타일, 레이아웃 구조, Workflow 구조, 대표 페이지 디자인은 변경하지 않았습니다.
- 표지/차례/Workflow 포스터/PART·Chapter 오프닝/이야기/프로젝트 화면/실습 페이지 스타일은
  6장 시안(v2.1)의 타이포그래피 수치를 그대로 사용합니다.

## 이번 단계에서 발견해 수정한 조판 결함(교정쇄 수준)

1. **원고 최상단 대표 제목 중복 노출** — `book-source-final.md` 1행의 `# 『보건교사를 위한 AI 업무 자동화』`가
   표지에 이미 쓰였음에도 차례 다음 페이지에 원문 그대로(`#` 포함) 다시 노출되던 결함. 렌더링 단계에서만
   건너뛰도록 수정(원고 삭제 아님).
2. **러닝 헤더(PART · CHAPTER 표시) 전체 누락** — 전체 조판 스크립트 구조상 페이지별 러닝 헤더가 문서 전체에
   걸쳐 전혀 표시되지 않던 결함(실제 렌더링 시점이 아니라 문단 생성 루프 종료 시점의 상태만 참조하던 버그).
   페이지가 실제로 그려지는 시점에 상태를 갱신하는 마커로 교체해 전체 250페이지에서 정상 표시되도록 수정.
3. **챕터 종료 직후 문장 1~2줄이 혼자 남는 현상** — 카드 묶음 뒤에 이어지는 짧은 마무리 문장이 다음
   페이지에 단독으로 남아 여백이 큰 페이지가 되던 2건(舊 실습 17/18 종료, 실습 21 종료 지점)을,
   승인된 시안에 이미 있던 "beat"(중앙 정렬 강조) 스타일로 렌더링해 의도된 마무리 문장처럼 보이도록 처리.
